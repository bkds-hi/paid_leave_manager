# 有給管理ツール 設計ドキュメント（Googleスプレッドシート版）

## 1. システム概要

Googleスプレッドシートで動作する有給管理ツール。日付と増減数を入力するだけで、Apps Scriptが自動的に残日数計算、付与日別管理、失効判定を行います。

---

## 2. シート構成（1シート）

| 列 | 項目 | 説明 | 入力/自動 |
|----|------|------|-----------|
| A | 日付 | 付与日または利用日 | 手動入力 |
| B | 増減数 | 正=付与、負=利用 | 手動入力 |
| C | 残日数 | その時点での総残日数 | 自動計算 |
| D | メモ | 任意の備考 | 手動入力（任意） |
| E | （空白） | 仕切り列 | - |
| F | 付与日 | 有給が付与された日 | 自動 |
| G | 付与日数 | 付与された日数 | 自動 |
| H | 使用済 | 使用した日数の合計 | 自動 |
| I | 残日数 | 付与日数 - 使用済 | 自動 |
| J | 失効予定日 | 付与日 + 2年 | 自動 |
| K | ステータス | 有効/期限間近/使用済/失効済 | 自動 |

---

## 3. 自動計算ロジック

### 3.1 残日数計算（C列）
```
残日数 = 前の行の残日数 + 今回の増減数
```

### 3.2 付与日別管理の更新（F-K列）
入力検知時に以下を自動実行：

**付与時（増減数 > 0）:**
- F-K列に新しい付与日を追加
- 付与日数 = 入力値
- 使用済 = 0
- 残日数 = 付与日数
- 失効予定日 = 付与日 + 2年
- ステータス = 「有効」

**利用時（増減数 < 0）:**
- 古い付与日から順に消費（FIFO方式）
- 各付与日の「使用済」を更新
- 各付与日の「残日数」を再計算
- ステータスを更新

### 3.3 ステータス判定
| 条件 | ステータス |
|------|-----------|
| 残日数 <= 0 | 使用済 |
| 失効予定日 <= 今日 | 失効済 |
| 失効予定日 <= 今日 + 30日 | 期限間近 |
| その他 | 有効 |

---

## 4. Apps Script構成

| 関数 | 役割 |
|------|------|
| `onEdit(e)` | 編集イベントのトリガー関数 |
| `autoProcess()` | 入力検知、残日数計算、一覧更新の呼び出し |
| `updateGrantSummary()` | F-K列（付与日別管理）を全面更新 |
| `recalculateAll()` | 手動再計算（メニューから実行） |
| `onOpen()` | メニュー追加 |

---

## 5. 入力フロー

```
ユーザーがA列（日付）とB列（増減数）を入力
    ↓
onEdit(e) トリガー発火
    ↓
autoProcess() 実行
    ├─ 残日数を全行再計算（C列）
    └─ updateGrantSummary() 呼び出し
        └─ F-K列を全面更新
```
